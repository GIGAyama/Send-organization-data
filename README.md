# Google Data Transfer

Google ドキュメント、スプレッドシート、スライド、および Google ドライブ上のファイルを、別のアカウント（個人アカウントなど）の Google ドライブへ直接コピー・移行するためのツールです。

## 概要

このツールは、組織（会社や学校）のアカウントにあるファイルを、組織外の個人アカウントへ簡単にバックアップ・移行することを目的としています。 共有制限により「オーナー権限の譲渡」ができない場合でも、ファイルをエクスポート・再アップロード・再変換するプロセスを自動化することで、物理的なコピーを作成し、制限を回避します。

## 特徴

*   **ワンクリック転送**: 現在開いているドキュメントやファイルを即座にコピーします。
    
*   **一括転送**: Google ドライブの画面で複数ファイルを選択し、まとめて処理できます。
    
*   **自動形式変換**: Google 形式のファイルを Office 形式（.docx, .xlsx, .pptx）でエクスポートし、コピー先で再び Google 形式に自動変換します。
    
*   **コンテキストメニュー対応**: 右クリックメニューから直接転送を開始できます。
    
*   **デスクトップ通知**: 転送の開始、進捗（○/○件）、完了をリアルタイムに通知。完了通知をクリックすると、コピー先のファイルをすぐに開けます。
    

## システム構成

1.  **Chrome 拡張機能**: ユーザーインターフェースと、ファイルのエクスポート・Base64 エンコードを担当。
    
2.  **Google Apps Script (GAS)**: 受け取り側アカウントで動作し、Drive API を用いたファイルの保存・変換を担当。
    

## インストール・セットアップ

### 1. 受け取り側（コピー先）アカウントの設定

1.  [Google Apps Script](https://script.google.com/ "null") にアクセスし、新しいプロジェクトを作成します。
    
2.  本リポジトリの `Code.gs` の内容をプロジェクトに貼り付けます。
    
3.  エディタ上の `setupAuth` 関数を一度実行し、ドライブへのアクセス権限を承認します。
    
4.  「デプロイ」 > 「新しいデプロイ」を選択します。
    
    *   **種類の選択**: ウェブアプリ
        
    *   **次のユーザーとして実行**: 自分
        
    *   **アクセスできるユーザー**: 全員（※GAS の URL を知っている人だけがアクセス可能です）
        
5.  発行された **ウェブアプリ URL** をコピーします。
    

### 2. Chrome 拡張機能の設定

1.  本リポジトリをダウンロードまたはクローンします。
    
2.  `background.js` の 2 行目にある `GAS_URL` の値を、先ほどコピーしたウェブアプリ URL に書き換えます。
    
3.  Google Chrome で `chrome://extensions/` を開きます。
    
4.  右上の「デベロッパー モード」をオンにします。
    
5.  「パッケージ化されていない拡張機能を読み込む」をクリックし、本プロジェクトのフォルダを選択します。
    

## 使い方

### 単一ファイルの転送

*   コピーしたい Google ドキュメント等のページを開いた状態で、拡張機能アイコンをクリックし、「開いているファイルを転送」を押します。
    

### 一括転送

*   Google ドライブのリスト画面で、転送したいファイルを 1 つまたは複数選択（青色反転）します。
    
*   拡張機能アイコンから「選択中のファイル一括転送」をクリック、または右クリックメニューの「個人アカウントへ転送」を選択します。
    

## 注意事項

*   **API 制限**: 短時間に大量のファイルを転送すると、Google ドライブ API の制限（Rate Limit）にかかる場合があります。その際は時間を置いてから再度実行してください。
    
*   **ファイルサイズ**: ブラウザのメモリ制限や GAS の実行時間制限により、非常に大容量のファイルは転送に失敗することがあります。
    
*   **セキュリティ**: GAS のウェブアプリ URL は、外部に漏れないよう厳重に管理してください。
